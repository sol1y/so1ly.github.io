How to use Github

Using Github is hard at first and can be very intimidating. Do not fret because this text will take you through how to use it! Firstly you want to visit github.com and sign into your account (Or create one). Now you are in! Github is a free to use programming tool which allows you to share, edit, team up and publish code with just you or other people! It is crucial to understand how to use Github and how everything runs, which is why I will try to explain it. There is a main live repository are which holds the latest published code. So for example, you publish a code you worked on, this code will go straight up to the live repository for others to access. Now there is a local repository. This is where YOUR code is held. Other contrubutors do not have access to your local repository, so it is just on YOUR computer/device. There is an action called fetching, this is where your computer fetches the live repository from the cloud and overides any local repository in your computer. So essentially it pulls down the most recent code that has been published from your team. Now there is commiting and pushing. These are two different things. Commiting is just holding it onto your computer and getting it ready for pushing, and pushing pushed this commited code upto the live repository and overrides the live repository with your code you published. There is so much more to Github, but this was just the basics!
